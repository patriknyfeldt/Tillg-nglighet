TILLGÄNGLIGHET

Vad betyder tillgänglighet?

Begreppet tillgänglighet innebär att man strävar efter att så många som möjligt ska kunna ta del av det som samhället har att erbjuda. Det kan handla om att alla ska kunna ta sig fram på till exempel vägar, trottoarer och i parker, kunna passera ett övergångsställe säkert eller resa med kollektivtrafiken även om man har någon funktionsnedsättning. Det innebär också att man ska kunna tillgodose sig information och kommunikation, så som media eller nyheter, eller kunna ta del av produkter eller tjänster. Idag sker mer och mer av våra liv digitalt, vilket kan förenkla väldigt mycket för personer som till exempel har svårt att ta sig ut från hemmet, så för att få ett inkluderande samhälle, måste vi göra våra tjänster/sajter tillgängliga.

Tillgänglighet på webben

Det finns otroligt mycket man kan och bör göra när man designar en webbsida eller e-tjänst, för att försöka göra den så tillgänglig som möjligt. Idag finns många olika hjälpmedel för folk med funktionsnedsättning, som till exempel skärmläsare, röststyrning och olika verktyg för att navigera sig på webbplatser. För att dessa hjälpmedel ska fungera som det är tänkt, måste vi utforma sidorna på ett tillgängligt sätt. Detta kan vi åstadkomma genom att använda oss av rätt semantiska element, märka upp vår HTML och se till att strukturen på sidan är lättbegriplig och tydlig. Det är även viktigt att se till att viktiga funktioner på sidan är lättåtkomliga, använda tydliga kontraster samt att göra sidan anpassningsbar (göra det möjligt att höja/sänka volym, förstora textstorlek eller ändra ljusstyrka).

Vad är webbtillgänglighet och varför är det viktigt?

Exempel på hur man skapar tillgänglig design på webben

  • Använd rätt semantiska HTML-taggar för rätt innehåll. Genom att använda oss av element som till exempel '<header>, <main>, <section>, <article>, <aside>, <footer>, gör vi det lättare för personer som använder hjälpmedel, exempelvis skärmläsare att navigera sig på sidan.
  • Gör det möjligt att anpassa sidan efter behov. Att kunna sänka/höja volymen, förstora texten eller ändra ljuststyrka kan underlätta mycket för personer med funktionsvariationer.
  • Se till att det är tydliga kontraster på sidan. Svaga kontraster kan göra det svårt för en person som är färgblind att se innehållet ordentligt.
  • Använd alt-attribut i bilder så att skärmläsare kan läsa upp vad en bild beskriver för användaren, även om man inte kan se den. Detta hjälper även sökmotorer att förstå vad bilden innehåller, och är bra för alla användare om bilden av någon anledning inte skulle vara tillgänglig.
  • Använd rätt språk på sidan. Om innehållet på sidan är på svenska, men en skärmläsare ”tror” att den är på engelska, kan uppläsningen bli väldigt fel.

Checklista för tillgänglig webbplats

Att tänka på när du skapar en tillgänglig webbplats

  • Använd beskrivande länktext. Om en länktext består av en url så kommer en skärmläsare läsa upp den url:en , och om en länktexten består av till exempel ”klicka här”, så kan det bli svårt för en person som använder skärmläsare att förstå vilken länk som leder vart. Använd istället en länktext som beskriver vart man hamnar om man trycker på den.
  • Om en bild är helt irrelevant för att förstå eller uppleva sidan, så kan man använda ett tomt alt-attribut (alt=””), för att förhindra att en skärmläsare läser upp bilden och förvirrar användaren i onödan.
  • Om det inte finns något semantiskt HTML-element som passar för att beskriva elementet, så bör man använda aria-attribut för att beskriva elementets roll.
  • Se till att det går att navigera sig på sidan med hjälp av tangentbordet på ett logiskt sätt.
  • Se till att viktiga knappar eller liknande är lätta att upptäcka och använda.
